# Changelog

**This `CHANGELOG/` directory is this repository's home for historical content.**
Every repository in the project keeps its own `CHANGELOG/` — history is
repo-scoped, never centralized in one file — split into one file per calendar
month so no single file grows without bound. `CLAUDE.md` and the skills describe
the CURRENT state in present tense; dated history, past renames, completed
cutovers, and retired identifiers live here and nowhere else. This directory is
the sanctioned "changelog context" named by CLAUDE.md R5's grep self-test.

## Layout

- One file per calendar month: `YYYY-MM.md` (e.g. `2026-06.md`). Entries within a
  file are reverse-chronological (`YYYY-MM-DD` stamps). New entries are appended
  to the current month's file — create it on the month's first entry.
